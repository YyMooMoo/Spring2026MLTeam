Q1. z = w.x + b = (3)(2) + (-2)(1) + (1)(4) + (-1)
    z = 6 - 2 + 4 - 1 = 7
    ReLU(z) = max(0, 7) = 7

Q2. Weight parameters only (biases ignored):
    Layer 1 (784 -> 128):  784 * 128 = 100,352
    Layer 2 (128 -> 64):   128 *  64 =   8,192
    Layer 3 (64  -> 10):    64 *  10 =     640
    Total = 109,184 weights

Q3. A linear layer is f(x) = W x. Stacking ten of them:
       y = W10 (W9 (... (W1 x) ...)) = (W10 W9 ... W1) x = M x
    The product of matrices is itself a matrix, so any number of stacked
    linear layers is mathematically equivalent to a single linear map.
    Activation functions (ReLU, sigmoid, tanh) inject nonlinearity between
    layers, which is what prevents the collapse and lets depth actually
    add representational power.

Q4. Problem: overfitting (low training loss, high validation loss -- the
    model memorized the training set instead of learning generalizable
    features).
    Two fixes (any two of the following):
      - Dropout: randomly zero activations during training so the network
        cannot rely on any single neuron.
      - L2 regularization (weight decay): penalize large weights to keep
        the model simpler.
      - Reduce model capacity: fewer layers or fewer hidden units.
      - Early stopping: halt training when validation loss starts rising.
      - More training data or data augmentation.


SECTION 2 -- CONVOLUTIONAL NEURAL NETWORKS
-----------------------------------------

Q5. Apply filter W = [1, 0, -1] with stride 1 to X = [3, 1, 4, 1, 5, 9].
    pos 0: (1)(3) + (0)(1) + (-1)(4) = 3 - 4 = -1
    pos 1: (1)(1) + (0)(4) + (-1)(1) = 1 - 1 =  0
    pos 2: (1)(4) + (0)(1) + (-1)(5) = 4 - 5 = -1
    pos 3: (1)(1) + (0)(5) + (-1)(9) = 1 - 9 = -8
    Output: [-1, 0, -1, -8]
    Pattern: this filter computes (x_i - x_{i+2}), so it responds to
    differences between values two steps apart. Large negative outputs
    mean a sharp rise in the signal -- it's an edge / gradient detector.

Q6. Spatial size at each step (input 64x64):
    1) Conv 3x3, stride 1, no padding: (64 - 3) / 1 + 1 = 62  ->  62 x 62
    2) MaxPool 2x2, stride 2:           62 / 2 = 31           ->  31 x 31
    3) Conv 3x3, stride 1, no padding: (31 - 3) / 1 + 1 = 29  ->  29 x 29

Q7. A neuron's "receptive field" is the region of the original image that
    can affect its output. In layer 1, each neuron sees only a 3x3 patch
    of pixels -- just enough information to detect an edge or local
    intensity change. A neuron in layer 2 sees a 3x3 patch of layer-1
    activations, which corresponds to roughly a 5x5 region of the
    original image. Each layer's effective receptive field grows.
    Deeper neurons therefore integrate information from progressively
    larger regions of the image, so they can represent combinations of
    edges (textures), then combinations of textures (parts), then
    parts (objects like 'face' or 'wheel').

Q8. Fully connected (64x64 -> 1000):
       (64 * 64) * 1000 = 4096 * 1000 = 4,096,000 weights
    Convolutional (32 filters, 3x3, 1 input channel):
       32 * 3 * 3 * 1 = 288 weights
    The conv layer has roughly 14,000x fewer parameters because each
    3x3 filter is reused (parameter sharing) at every spatial position
    in the image. This means much less memory, faster training, less
    overfitting, and the same filter can detect a feature anywhere in
    the image (translation-equivariant).


SECTION 3 -- TRANSFORMERS & ATTENTION
------------------------------------

Q9. Attention computes the embedding for "with" as a weighted sum of all
    other tokens' values, where the weights come from how well each
    token's key matches the query for "with". The sentence is genuinely
    ambiguous:
      - Instrument reading ("I used a telescope to see the man"): the
        attention head puts more weight on "saw" + "telescope".
      - Modifier reading ("the man who had a telescope"): the head puts
        more weight on "man" + "telescope".
    Different attention heads can settle on different parses, so the
    model can represent both interpretations simultaneously. This is one
    reason multi-head attention is useful, and it is something an RNN's
    single hidden state cannot do because it would compress the prior
    context into a single vector.

Q10. Q = [1, 0],  K1 = [1, 0],  K2 = [0, 1],  K3 = [-1, 0],  d_k = 2.
     Raw scores: Q.K1 = 1,  Q.K2 = 0,  Q.K3 = -1
     Scaled by 1/sqrt(2):  [0.7071,  0.0000, -0.7071]
     exp:                 [2.0281,  1.0000,  0.4931]   sum = 3.5212
     Softmax:             [0.5761,  0.2840,  0.1400]
     Q attends most strongly to K1 = [1, 0] -- same direction as Q, so
     the dot product (and therefore the score) is largest.

Q11. Causal mask for the first 4 tokens. Rows = query token, cols = key
     token. 1 = can attend, 0 = masked (set to -inf before softmax).
     Sentence: "I love to eat french fries with ketchup."

                  I   love   to   eat
        I      [  1     0     0    0  ]
        love   [  1     1     0    0  ]
        to     [  1     1     1    0  ]
        eat    [  1     1     1    1  ]

     Each token attends to itself and all earlier tokens only; future
     tokens are masked out so the model cannot peek ahead during
     autoregressive prediction. The full 9x9 mask for the whole
     sentence is just the lower-triangular extension of this pattern.

Q12. BERT -- bidirectional attention: every token can attend to every
     other token. Best for understanding tasks: classification, named-
     entity recognition, span extraction, question answering. Cannot
     generate left-to-right because at every position it sees the
     answer.
     GPT -- causal (left-to-right) attention: each token attends only
     to past tokens. Best for generation tasks: text completion, chat,
     summarization. The masking choice directly enables generation:
     at inference time you only have past tokens, so masking future
     ones during training matches the actual generation setting.


SECTION 4 -- REINFORCEMENT LEARNING
----------------------------------

Q13. gamma = 0.8, rewards = [0, 0, 5, 0, 10].
     G_0 = r0 + g*r1 + g^2*r2 + g^3*r3 + g^4*r4
         = 0 + 0.8(0) + 0.64(5) + 0.512(0) + 0.4096(10)
         = 0 + 0 + 3.2 + 0 + 4.096
         = 7.296

Q14. Markov property: the next state depends only on the current state
     and current action, not on the full history of how we got here.
     Formally: P(s_{t+1} | s_t, a_t) = P(s_{t+1} | s_0, a_0, ..., s_t, a_t).
     Holds: chess. The board position fully captures the state of the
     game; how you reached this position does not change the legal moves
     or the optimal play from here.
     Does NOT hold: poker with hidden hole cards (history of betting and
     unseen cards influences inferences), or medical diagnosis from a
     single snapshot (prior history matters), or a stock price (past
     trends carry information beyond the current price).

Q15. Credit assignment problem: when reward arrives only at the end of a
     long sequence of decisions, how do you figure out which earlier
     decisions actually caused the outcome?
     Chess analogy: you win on move 40 with reward +1. Your moves 1
     through 39 each had immediate reward 0. Which of those early moves
     actually led to the win, and which were neutral or mistakes that
     happened to work out?
     Bellman equation: V(s) = E[r + gamma * V(s')]. Each state's value
     is defined recursively in terms of the next state's value, so the
     terminal reward propagates one step backward per update. Repeated
     updates push value (credit) from the goal back to earlier states,
     so good early moves eventually get high value because they lead to
     high-value successor states.

Q16. epsilon = 0.1.
     (a) 10% of actions are random (the rest are greedy).
     (b) Assume 4 possible actions and one of them steps off the cliff
         for -100. A random action picks the cliff with probability 1/4.
         Expected penalty per step from exploration alone:
            0.1 * (1/4) * (-100) = -2.5 per step
     (c) epsilon should be HIGHER early in training. Early on the agent
         knows nothing, so exploration is valuable -- random actions
         gather information. Late in training the policy is already
         good and random actions just hurt performance, so epsilon
         should be small. Common practice is epsilon-decay: start
         around 1.0 and anneal down to ~0.01 over training.


SECTION 5 -- Q-LEARNING
----------------------

Q17. Q(s,a) = 4, r = 8, Q(s', left) = 6, Q(s', right) = 10,
     alpha = 0.3, gamma = 0.9.
     max_{a'} Q(s', a') = max(6, 10) = 10
     TD target = r + gamma * max Q(s', a') = 8 + 0.9 * 10 = 17
     TD error  = target - Q(s,a) = 17 - 4 = 13
     New Q(s,a) = Q(s,a) + alpha * TD error
                = 4 + 0.3 * 13
                = 4 + 3.9
                = 7.9

Q18. 4-state chain S0 -> S1 -> S2 -> S3 (terminal).
     r(S2 -> S3) = 20, all other rewards 0. alpha = 0.5, gamma = 1.0.
     All Q values initialized to 0. Update:
        Q(s,a) <- Q(s,a) + alpha * [ r + gamma * max Q(s', .) - Q(s,a) ]

     Step                 Trans   r    Update                              Q(S1)   Q(S2)
     Ep1 Step1 (S0->S1)   S0->S1  0    Q(S0) <- 0 + 0.5[0 + 0 - 0] = 0        0       0
     Ep1 Step2 (S1->S2)   S1->S2  0    Q(S1) <- 0 + 0.5[0 + 0 - 0] = 0        0       0
     Ep1 Step3 (S2->S3)   S2->S3  20   Q(S2) <- 0 + 0.5[20 + 0 - 0] = 10      0      10
     Ep2 Step1 (S0->S1)   S0->S1  0    Q(S0) <- 0 + 0.5[0 + 0 - 0] = 0        0      10
     Ep2 Step2 (S1->S2)   S1->S2  0    Q(S1) <- 0 + 0.5[0 + 10 - 0] = 5       5      10
     Ep2 Step3 (S2->S3)   S2->S3  20   Q(S2) <- 10 + 0.5[20 + 0 - 10] = 15    5      15

     Key takeaway: credit propagates back one state per episode. Q(S2)
     learns about the reward in episode 1, Q(S1) learns about Q(S2) in
     episode 2, and Q(S0) would learn about Q(S1) in episode 3. With
     gamma = 1 and enough episodes, every Q value converges to 20.

Q19. Off-policy: the policy used to generate experience (the BEHAVIOR
     policy, e.g. epsilon-greedy) is allowed to differ from the policy
     being learned (the TARGET policy, the greedy one). Q-learning
     always updates toward "max over next actions," regardless of what
     action the agent actually took next.
     Q-learning update:
        Q(s,a) <- Q(s,a) + alpha * [ r + gamma * max_{a'} Q(s', a') - Q(s,a) ]
     SARSA update:
        Q(s,a) <- Q(s,a) + alpha * [ r + gamma * Q(s', a')          - Q(s,a) ]
     Key difference: Q-learning uses max Q(s', a') (the best possible
     next action). SARSA uses Q(s', a') where a' is the ACTUAL next
     action taken by the current policy. SARSA is on-policy -- it
     accounts for the fact that the agent will keep exploring, so it
     learns a "safer" policy near cliffs (e.g. it stays farther from
     the cliff edge because it expects future random actions).

Q20. Q-table size: 10,000 states * 5 actions = 50,000 entries.
     Works fine: small, discrete environments -- grid worlds, simple
     board games, basic discrete-action robot tasks. 50,000 entries is
     trivially small to store and update.
     Completely unworkable: high-dimensional or continuous state spaces.
     Atari pixels (210 x 160 x 3 with 256 levels per channel) give on
     the order of 10^60+ possible states -- more than atoms in the
     universe. You can't enumerate them, store them, or visit each one
     enough times to learn its value.
     Standard fix: function approximation with a neural network -- Deep
     Q-Network (DQN). Replace the table Q[s][a] with a network
     Q(s, a; theta) that takes the raw state (e.g. pixels) as input
     and outputs Q-values for each action. The network generalizes
     across similar states instead of memorizing each one.
